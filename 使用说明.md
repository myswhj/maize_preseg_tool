# 玉米标注工具使用说明

本文档面向实际使用者，按当前程序真实行为说明：

- 如何手工标注
- 如何标记“已完成”
- 如何触发训练
- 如何查看和使用预标注候选层
- 如何导出和回退模型

当前入口文件是 `main.py`。

## 1. 安装与启动

安装依赖：

```bash
pip install -r requirements.txt
```

启动 GUI：

```bash
python main.py
```

如果只使用主流程，可以暂时不管 SAM；缺少 `segment-anything` 只影响 SAM 功能，不影响标注、训练、预标注和导出。

## 2. 第一次进入项目

程序按“图片公共目录”自动识别项目。

你加载一批图片后，程序会自动为这批图片进入或创建一个项目。每个项目都有自己的：

- 标注记录
- 图片状态
- 固定验证集清单
- 模型版本目录
- active 模型

项目目录位于：

```text
maize_annotations/project_state/<project_id>/
```

## 3. 这版工具里的几个关键概念

### 3.1 正式实例

正式实例是你已经确认的标注：

- 会保存进 `.maize`
- 会进入训练集
- 会参与导出

### 3.2 候选实例

候选实例是 AI 预标注给出的建议：

- 只显示在候选层
- 不会自动保存
- 必须先“接受候选”才会变成正式实例

### 3.3 类别

类别是训练真正使用的标签，比如：

- `plant`
- `stem`
- `leaf`
- `ear`

### 3.4 所属植株组

所属植株组只用于交互管理和追溯：

- 用于右侧树分组显示
- 用于人工归属管理
- 不会进入 YOLO 训练标签

### 3.5 已完成图片

只有标记为“已完成”的图片才会：

- 纳入训练集
- 参与自动训练计数
- 进入项目级 YOLO 导出

如果一张已完成图片之后又被修改，它会重新计入 `dirty completed`。

## 4. 界面怎么理解

### 4.1 左侧区域

左侧主要是旧工具栏，负责：

- 边缘吸附
- 暂存区域
- 保存整株
- 删除实例
- 图片切换
- SAM / 区域生长入口

### 4.2 中间双画布

- 左画布：主编辑画布，显示正式层、候选层、当前手工绘制内容
- 右画布：正式实例总览，不显示候选层

### 4.3 右侧属性面板

右侧主要看这几块：

- 项目状态
- Active 模型
- 训练状态和进度
- 已完成图片数量
- Dirty Completed 数量
- 实例分组树
- 当前选中实例的类别、所属植株组、来源信息

## 5. 冷启动怎么用

一个新项目刚开始通常没有模型。

这时会出现：

- `Active 模型 = 暂无模型`
- 打开图片时不会自动预标注

你需要先手工标注若干张图，作为冷启动数据。

推荐顺序：

1. 批量加载同一目录下的图片
2. 手工标注
3. 每张确认后点击“标记当前图片为已完成”
4. 当 `dirty completed >= 5` 时，自动触发第一次后台训练
5. 第一次训练成功后，再打开未标注图片才会自动出现候选层

右侧“手动启动训练”按钮可用于调试或兜底，不要求一定达到 5 张。

## 6. 手工标注怎么做

### 6.1 画 polygon

在左侧主画布中：

- 鼠标左键：逐点加顶点
- 鼠标右键拖动：平移
- 滚轮：缩放
- `Shift`：切换边缘吸附

### 6.2 暂存一个区域

画完一个区域后按：

```text
Enter
```

这会把当前 polygon 暂存到“当前实例预览层”。

### 6.3 保存一个正式实例

当一个实例的所有区域都准备好后按：

```text
Shift+Enter
```

这会把当前预览层保存为一个正式实例。

注意：

- 一个正式实例可以包含多个 polygon
- 如果最后一个区域刚画完，还没先按 `Enter`，直接按 `Shift+Enter` 也可以，程序会自动先暂存当前区域再保存整株

### 6.4 去除区域

去除区域用于在主多边形中挖空部分区域：

- 按 `R` 进入去除区域绘制模式
- 在主多边形内部绘制要去除的区域
- 按 `Enter` 暂存去除区域
- 保存整株时，去除区域会自动从主多边形中挖空
- 去除区域在绘制时显示为透明效果，露出底层图像

### 6.5 撤销

```text
Ctrl+Z
```

可用于撤销：

- 当前绘制中的点
- 当前暂存区域
- 最近新增/删除的正式实例

### 6.6 删除实例

删除方式：

- 在右侧实例树选中后点击“删除当前选中实例”
- 在左侧实例下拉框中选中后删除
- 直接按 `Delete`

## 7. 类别和所属植株组怎么改

选中一个正式实例或候选实例后，可以在右侧属性面板修改：

- 类别
- 所属植株组

如果要新建植株组：

- 选中实例
- 点击“新建植株组并绑定”

说明：

- 候选实例改类别后，接受时会按新的类别转正式
- 正式实例如果原本来自 AI，后续被你手工改点、改类别或改归属，`source` 会转成 `ai_modified`

## 8. 预标注候选层怎么用

### 8.1 自动预标注触发条件

程序只会在以下条件同时满足时自动预标注：

- 当前项目已有 active 模型
- 当前图片没有正式实例
- 当前图片不是“已完成”

如果当前图片已经有正式实例，程序默认不会自动覆盖。

### 8.2 手动执行 AI 预标注

右侧有一个按钮：

- `对当前图执行AI预标注`

它的作用是：

- 手动对当前图再跑一次预标注
- 即使当前图片已有正式实例，也能查看候选层
- 不会覆盖正式实例

这适合做两件事：

- 调试当前模型到底有没有检出
- 在已有人工标注的图上做对照参考

### 8.3 候选层显示在哪里

- 候选层只显示在左侧主画布
- 右侧总览画布不显示候选层
- 右侧实例树里会出现 `Candidates` 分组

### 8.4 接受或清理候选

候选支持：

- 接受当前候选
- 接受全部候选
- 清空候选
- 删除当前候选

接受后：

- 会转成正式实例
- `source = ai_accepted`
- 会记录来源模型版本和置信度

如果你后面又改它：

- 会转成 `ai_modified`

### 8.5 如果点了预标注但没看到候选

这不一定是 GUI 出错。

程序现在会明确显示：

- `预标注完成: N 个候选`
- 或 `预标注完成: 0 个候选（当前模型未检出）`

如果是 `0 个候选`，说明这次推理本身就没有检测结果，不是“画布没显示出来”。

## 9. 已完成状态怎么用

一张图确认无误后：

- 点击“标记当前图片为已完成”

如果想撤回：

- 点击“取消当前图片已完成”

“已完成”会持久化保存，不只是内存状态。

## 10. 自动训练规则

### 10.1 什么时候会自动训练

每次保存一张已完成图片后，程序都会检查：

- 自上次成功训练以来，`dirty completed` 是否达到 `5`

达到后：

- 自动后台启动一次训练

### 10.2 训练用哪些数据

每轮训练都使用：

- 当前项目下
- 所有“已完成”图片
- 当前最终正式标注

不会保留旧实例残留；数据集每次都是按当前正式标注重建。

### 10.3 训练不会卡死 GUI

训练由独立脚本在独立进程中运行，不在 GUI 主线程里执行。

训练过程中，右侧会显示：

- 训练状态文字
- 训练进度条
- 当前 active 模型版本

### 10.4 训练成功和失败分别会怎样

训练成功：

- 自动 promote 新模型为 active
- 后续预标注使用新模型

训练失败：

- 保留旧模型继续使用
- 不会回滚或破坏已确认标注

## 11. 固定验证集

首次构建训练集时：

- 默认 `val_ratio = 0.2`
- 默认最小 `val = 1`

后续：

- 原有 `split_manifest.json` 不会自动变化
- 新增已完成图片默认进入 `train`

如果你想重建验证集：

- 点击“重建验证集”

重建后，程序会重新生成固定的 `train / val` 划分。

## 12. 模型版本和回退

### 12.1 查看当前 active 模型

右侧“项目状态”会显示：

- `Active 模型`

版本目录一般位于：

```text
maize_annotations/project_state/<project_id>/models/versions/model_v0001/
```

### 12.2 回退到上一模型

如果你觉得新模型效果更差：

- 点击“回退到上一模型”

回退只影响：

- 后续预标注

不会影响：

- 已保存正式标注

## 13. 导出功能

### 13.1 当前图片导出

支持：

- 导出当前 JSON
- 导出当前 COCO

### 13.2 项目级 YOLO 数据集导出

点击：

- `导出项目YOLO数据集`

程序会在项目目录下重建：

```text
dataset/
├─ images/train
├─ images/val
├─ labels/train
├─ labels/val
└─ data.yaml
```

导出规则：

- 只导出正式实例
- 只导出已完成图片
- 只导出类别和 polygon
- 不导出所属植株组信息

### 13.3 批量导出已完成(coco格式)

点击：

- `批量导出已完成(coco格式)`

会导出项目下已完成图片对应的 COCO 格式 JSON 文件，包含标注信息和忽略区域。

## 15. 推荐操作顺序

1. 批量加载同一项目目录下的图片
2. 冷启动阶段先手工标注
3. 每张确认后点击“标记当前图片为已完成”
4. 达到 `dirty completed >= 5` 后等待后台训练
5. 训练成功后打开未标注图，观察自动候选层
6. 必要时点击“对当前图执行AI预标注”做手动复查
7. 接受候选、修正候选或补画漏检
8. 再次标记该图为已完成
9. 随着 dirty completed 累积，触发下一轮训练

## 16. 常见问题

### 16.1 为什么打开图片没有自动候选层？

常见原因：

- 当前项目还没有 active 模型
- 当前图片已经有正式实例
- 当前图片已经标记为已完成

### 16.2 为什么手动点了 AI 预标注还是没有显示？

请先看右侧状态文字。

如果显示：

- `预标注完成: 0 个候选（当前模型未检出）`

说明这次推理确实没有检出，不是候选层没画出来。

### 16.3 为什么训练没有自动开始？

请检查：

- 是否真的标记了“已完成”
- `dirty completed` 是否达到 `5`
- 当前是否已有训练在运行

如果只是临时想试跑：

- 直接点“手动启动训练”

### 16.4 为什么训练没走显卡？

这通常不是 GUI 逻辑问题，而是当前 Python 环境里的 `torch` 不是 CUDA 版。

程序训练前会输出 `torch / CUDA` 诊断；如果你要求 GPU 但当前环境不支持 CUDA，会直接给出明确报错。

### 16.5 植株组归属有什么用？

它只用于：

- 交互分组
- 列表查看
- 数据追溯

第一版训练不会使用这个字段。

## 17. 当前建议

如果你要先把整套闭环跑通，建议优先走这条路径：

- 手工标注
- 标记已完成
- 自动训练
- 自动/手动预标注
- 候选接受和修正
- 再训练

SAM 和区域生长当前更适合作为辅助入口，而不是主流程依赖。
